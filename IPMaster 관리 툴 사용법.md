
# 1. 차단 및 원복  
## 작업 목적  
  
![](./Attached%20files/Pasted%20image%2020260430163342.png)  
  
차단 및 원복 작업의 목적은 하나의 도메인에 여러 IP가 연결되어 있을 때, 유지보수 중인 서버로 접근하지 않도록 특정 IP를 응답 대상에서 제외하는 것입니다.  
  
즉, 한 도메인에 속한 두 개 이상의 IP 중 한쪽 서버가 유지보수 중이라면, 정상 운영 중인 서버의 IP만 응답하도록 설정하여 유지보수 중인 서버로의 접근을 차단합니다.  
  
![](./Attached%20files/Pasted%20image%2020260430165627.png)  
  
위 화면처럼 도메인에 속한 각 그룹을 확인하면, `VIP Group` 아래에 IP가 표시됩니다. 이 IP가 해당 도메인에 대해 DNS 쿼리를 보냈을 때 응답으로 반환되는 값입니다.  
  
예를 들어 `a.test.com` 도메인은 다음 두 IP를 응답값으로 가집니다.  
  
- `121.254.214.24`
- `121.125.66.200`
  
왼쪽 중단의 `발신자 IP` 항목에 임의의 값을 입력한 뒤 쿼리를 보내면 IP 응답값을 확인할 수 있습니다.  
  
`POP Group Policy`가 `RoundRobin By Query`로 설정되어 있고, 각 그룹의 가중치가 `1`인 경우, 쿼리를 보낼 때마다 IP가 번갈아 응답됩니다. 이는 Load Balancing을 위한 설정입니다.

## 작업 전 준비  
  
![](./Attached%20files/Pasted%20image%2020260430200507.png)  
  
![](./Attached%20files/Pasted%20image%2020260430195625.png)  
  
작업 전에는 반드시 현재 설정값을 스크린샷으로 남겨야 합니다.  
  
특히 다음 내용을 정확히 확인해야 합니다.  
  
- 어떤 도메인에서 작업하는지  
- 어떤 그룹을 제외할 것인지
- 각 그룹의 가중치가 몇으로 설정되어 있는지
  
도메인을 클릭하면 오른쪽 상단에서 `POP Group Policy` 값을 확인할 수 있습니다.  
  
차단 후 다시 원복하면 가중치가 기본값인 `1`로 변경될 수 있습니다. 따라서 원복 후에는 반드시 기존 가중치와 동일하게 다시 수정해야 합니다.  
  
---  
  
## 작업 방법  
  
### 차단  
  
![](./Attached%20files/Pasted%20image%2020260430163436.png)  
  
차단하려는 그룹을 우클릭하면 위 화면처럼 `Include`, `Exclude` 메뉴가 표시됩니다.  
  
차단하려는 그룹에서 `Exclude`를 선택한 뒤, 왼쪽 상단의 `확인` 버튼을 눌러 적용합니다.  
  
적용 후 다시 쿼리를 보내면, 제외된 그룹의 IP는 더 이상 응답되지 않고 현재 포함되어 있는 그룹의 IP만 응답됩니다.  
  
---  
  
### 원복  
  
![](./Attached%20files/Pasted%20image%2020260430200755.png)  
  
원복할 때는 도메인을 우클릭한 뒤 `Include` → `Group`을 선택합니다.  
  
![](./Attached%20files/Pasted%20image%2020260430200832.png)  

그러면 포함할 그룹을 선택할 수 있는 화면이 나타납니다.
  
테스트 도메인은 그룹 수가 적어 쉽게 찾을 수 있지만, 실제 요청으로 들어오는 도메인은 많은 그룹을 가지고 있을 수 있습니다. 따라서 작업 전에 저장해 둔 스크린샷을 확인하여 정확히 어떤 그룹을 다시 포함해야 하는지 확인해야 합니다.

그룹 목록은 알파벳 순으로 정렬되어 있습니다.

이전에 차단했던 그룹을 선택한 뒤 `확인` 버튼을 눌러 포함합니다. 이후 다시 한 번 왼쪽 상단의 `확인` 버튼을 눌러야 그룹 포함이 최종 적용됩니다.  

![](./Attached%20files/Pasted%20image%2020260430195625.png)  
  
그룹을 다시 추가한 뒤에는 가중치 등 다른 설정값이 원래 값과 동일한지 반드시 확인해야 합니다.  

가중치가 기존 값과 다를 경우, 값을 수정한 뒤 최하단의 `확인` 버튼을 눌러 적용해야 합니다.  

이 화면에는 `확인` 버튼이 총 두 개 있으므로, 어떤 설정을 적용하는 버튼인지 잘 확인해야 합니다.

## 확인  
  
절차대로 작업했음에도 정상적으로 원복되지 않는 경우가 있을 수 있습니다. 이때는 유지보수가 끝난 서버가 정상적으로 연결되어 있는지 먼저 확인해야 합니다.  
  
상단 메뉴바에서 `서버 모니터`로 들어갑니다.  

![](./Attached%20files/Pasted%20image%2020260430201339.png)  
  
![408](./Attached%20files/Pasted%20image%2020260430201355.png)  
  
서버 모니터에서 초록불은 연결이 정상적으로 되어 있다는 의미입니다.  
  
반대로 유지보수가 아직 끝나지 않았거나 서버 연결에 문제가 있는 경우, 해당 서버가 붉은색으로 표시됩니다.  
  
이외의 문제가 발생하면 김상연 차장님께 즉시 연락해야 합니다.

## 종료  
  
![](./Attached%20files/Pasted%20image%2020260430163342.png)  
  
![](./Attached%20files/Pasted%20image%2020260430201942.png)  
  
작업을 종료할 때는 반드시 왼쪽 상단의 `X` 버튼을 눌러 서버 연결을 먼저 끊은 뒤 프로그램을 종료해야 합니다.  
  
서버 연결을 끊지 않고 프로그램만 종료하면 연결이 유지된 상태로 리소스를 계속 사용할 수 있습니다. 이 상태가 반복되면 하드웨어에 문제가 발생할 수 있으므로, 반드시 서버 연결을 먼저 종료해야 합니다.
